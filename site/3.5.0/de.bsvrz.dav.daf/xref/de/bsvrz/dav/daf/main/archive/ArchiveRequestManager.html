31   * Schnittstelle, die von Applikationen benutzt wird, um Anfragen an ein Archivsystem zu stellen. Eine konkrete Implementierung wird von den
32   * Datenverteiler-Applikationsfunktionen nach erfolgreichem Verbindungsaufbau zum Datenverteiler über die Methode {@link
33   * de.bsvrz.dav.daf.main.ClientDavInterface#getArchive} zur Verfügung gestellt.

35   * @author Kappich Systemberatung

40  	/**
41  	 * Spezifiziert die Größe des Empfangspuffers je Archivanfrage, die mit nachfolgenden <code>request</code> Aufrufen erzeugt werden. Der angegebene Parameter
42  	 * ist eine grobe Vorgabe für alle Streams einer Archivanfrage zusammen und muss von einer Implementierung nicht exakt berücksichtigt werden, sondern sollte in
43  	 * der sendeseitigen Blockbildung und in der Flusskontrolle so berücksichtigt werden, dass die Summe der vom Archivsystem gesendeten aber noch nicht von der
44  	 * anfragenden Applikation abgerufenen Bytes in der angegebenen Größenordnung liegt. Der Defaultwert ist 0 mit der Bedeutung, das die Größe durch einen
45  	 * entsprechenden Parameter des Archivsystems festgelegt wird.
46  	 *
47  	 * @param numberOfBytes Grobe Vorgabe bezüglich der Anzahl zu puffernden Bytes pro Archivanfrage oder 0, wenn keine Vorgabe seitens der Empfangsapplikation
48  	 *                      gemacht werden soll und statt dessen ein entsprechender Parameter des Archivsystems verwendet werden soll.
49  	 */
50  	void setReceiveBufferSize(int numberOfBytes);

98  	 * Die Implementation dieser Methode meldet sich als Empfänger von aktuellen Daten an. Die zurückgegebenen Datensätze unterscheiden sich von einer "normalen"
99  	 * Anmeldung als Empfänger, da Archivdaten zeitlich vor die aktuellen Daten gemischt werden. Der Benutzer legt dabei fest wie gross der Zeitraum sein soll, der
100 	 * vor die aktuellen Daten gemischt werden soll. Der Zeitraum wird als "Zeitspanne" oder als "Anzahl Datensätze" angegeben. </br> Ist kein Archivsystem
101 	 * vorhanden oder bei der Archivanfrage tritt ein Fehler auf, werden nur die aktuellen Daten zurückgegeben.
102 	 *
103 	 * @param receiver        Objekt, das Methoden zur Verfügung stellt um den Strom aus historischen und aktuellen Daten entgegen zu nehmen
104 	 * @param object          System-Objekt für die die spezifizierten Daten anzumelden sind
105 	 * @param dataDescription Beschreibende Informationen zu den anzumeldenden Daten
106 	 * @param options         Für die Anmeldung zu verwendende Optionen
107 	 * @param historyType     <code>HistoryTypeParameter.TIME</code> = Der Parameter <code>history</code> bezieht sich auf einen Zeitraum, der vor den aktuellen
108 	 *                        Daten liegen soll (in ms); <code>HistoryTypeParameter.INDEX</code> = Der Parameter <code>history</code> bezieht sich auf die Anzahl
109 	 *                        Datensätze, die mindestens vor den aktuellen Daten liegen sollen
110 	 * @param history         Zeitraum der Archivdaten in Millisekunden, die vor den ersten aktuellen Datensätzen liegen
111 	 */

115 			DataDescription dataDescription,
116 			ReceiveOptions options,
117 			HistoryTypeParameter historyType,
118 			long history);

197 	 * Beauftragt das Archivsystem die angegebenen Zeitbereiche ohne Rücksicht auf den mit {@link ArchiveRequestManager#increaseDeleteTime} festgelegten Wert zu
198 	 * löschen. Das Löschen der Zeitbereiche wird dabei in zwei Varianten unterteilt. In der ersten Variante werden die Zeitbereiche umgehend aus dem direkten
199 	 * Zugriff des Archivsystems entfernt. In der zweiten Variante werden die Zeitbereiche nur als "zu löschend" markiert, sobald das "automatische Löschen" des

204 	 * angegebene Zeitbereich nicht gelöscht.
205 	 *
206 	 * @param dataDisposedToDelete Zeitbereich(e), die gelöscht werden sollen
207 	 * @param deleteImmediately    true = Variante 1, die Zeitbereiche werden umgehend aus dem direkten Zugriff des Archivsystems entfernt; false = Variante 2, die
208 	 *                             Zeitbereiche werden nur als "zu löschend" markiert und später aus dem direkten Zugriff des Archivsystems entfernt
