29  /**
30   * Schnittstelle zum Zugriff auf die Eigenschaften eines System-Objekts. Datensätze, die vom Datenverteiler transportiert werden, sind immer genau einem
31   * System-Objekt zugeordnet. Zur internen Identifizierung eines System-Objekts wird die <code>id</code> des Objekts benutzt. Das ist ein 64-Bit-Wert der
32   * systemweit eindeutig ist. Zur expliziten Referenzierung von Objekten (z.B. in Aufrufparametern von Applikationen, Versorgungsskripten etc.) kann die
33   * permanente ID (PID) eines Objekts verwendet werden. Diese ist optional und muss nur bei den Objekten gesetzt werden, bei denen sie gebraucht wird. Die PID
34   * ist eine leicht zu merkende Zeichenkette, die systemweit eindeutig ist und sich (wenn sie einmal vergeben wurde) nie mehr ändern kann. Außerdem kann jedem
35   * System-Objekt ein Name zugeordnet werden, der zu Darstellungszwecken benutzt werden kann. Der Name eines Objekts kann i.a. jederzeit geändert werden und
36   * sollte aus diesem Grund nicht zur Referenzierung von Objekten eingesetzt werden.
37   *
38   * @author Kappich Systemberatung
39   * @version $Revision: 5084 $
40   */

46  	 * @return ID dieses System-Objekts oder 0, wenn das Objekt keine ID hat.
47  	 */
48  	public long getId();

51  	 * Liefert den Typ dieses System-Objekts zurück.

53  	 * @return Typ dieses System-Objekts.
54  	 */
55  	public SystemObjectType getType();

59  	 * zurückgeliefert wird. Außerdem ist das Objekt ein Element von den direkten und indirekten {@link SystemObjectType#getSuperTypes Super-Typen} des
60  	 * Objekt-Typs.
61  	 *
62  	 * @param type Zu prüfender Typ.
63  	 *
64  	 * @return <code>true</code>, wenn der übergebene Typ mit dem Objekt-Typ oder mit einem der direkten oder indirekten Vorgänger in der Vererbungshierarchie
65  	 *         übereinstimmt; sonst <code>false</code>.
66  	 */
67  	public boolean isOfType(SystemObjectType type);
68  
69  	/**
70  	 * Bestimmt, ob dieses System-Objekt ein Element des angegebenen Typs ist. Ein System-Objekt ist Element des Objekt-Typs, der von der Methode {@link #getType}
71  	 * zurückgeliefert wird. Außerdem ist das Objekt ein Element von den direkten und indirekten {@link SystemObjectType#getSuperTypes Super-Typen} des
72  	 * Objekt-Typs.
73  	 *
74  	 * @param typePid PID des zu prüfenden Typs.
75  	 *
76  	 * @return <code>true</code>, wenn der übergebene Typ mit dem Objekt-Typ oder mit einem der direkten oder indirekten Vorgänger in der Vererbungshierarchie
77  	 *         übereinstimmt; sonst <code>false</code>.
78  	 */
79  	public boolean isOfType(String typePid);
80  
81  	/**
82  	 * Liefert die permanente ID (PID) dieses Objekts zurück. Wenn das Objekt keine PID hat, wird ein leerer String zurückgegeben.

86  	public String getPid();

92  	 */
93  	public String getName();

133 	 */
134 	public String toString();

137 	 * Bestimmt, ob dieses Objekt zum aktuellen Zeitpunkt gültig ist. Konfigurierende System-Objekte sind nach deren {@link
138 	 * ConfigurationArea#createConfigurationObject Erzeugung} noch nicht gültig, sondern werden erst mit der Aktivierung der nächsten Konfigurationsversion gültig.
